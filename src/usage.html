使用说明与 Cloudflare API Token 配置指南

快速开始

  1. 按下方步骤创建 Cloudflare API Token。
  2. 打开应用，点击右上角 “添加账户”。
  3. 填写账户名称、邮箱（可选），粘贴 API Token。
  4. 点击 “验证并保存”，应用会自动获取 Account ID 并加载域名列表。

Cloudflare API Token 获取与权限设置

1

进入 API Token 页面

打开浏览器登录 Cloudflare Dashboard，点击下方链接：

https://dash.cloudflare.com/profile/api-tokens

点击 “Create Token” → 选择 “Custom token”（自定义令牌）。

2

推荐权限配置

根据你要管理的资源，在 Permissions 区域添加以下权限：

范围 资源 权限
帐户D1编辑
帐户Cloudflare Pages编辑
帐户Workers R2 存储编辑
帐户Workers KV 存储编辑
帐户Workers 脚本编辑
帐户帐户读取
帐户帐户设置读取
区域区域设置编辑
区域区域读取
区域防火墙服务编辑
区域Workers 路由编辑
区域DNS编辑
区域Analytics读取
帐户Cloudflare Tunnel编辑
精简版：如果只需要 域名 + DNS + 缓存 + 防火墙 + 分析，最少需要： 区域 - 区域（读取）、区域设置（编辑）、DNS（编辑）、防火墙服务（编辑）、Analytics（读取）， 帐户 - 帐户（读取）、帐户设置（读取）。
扩展版：如需使用本次新增的 Worker 自定义域名 / Zone Routes 或 Pages 自定义域名，请额外勾选： 帐户 - 帐户（读取）、Workers 脚本（编辑）、Cloudflare Pages（编辑）； 区域 - Workers 路由（编辑）、区域（读取）。
3

设置资源范围

Account Resources：选择你的 Cloudflare 账户。

Zone Resources：

  • Include > All zones —— 管理该账户下所有域名
  • Include > Specific zone —— 只管理指定域名
4

创建并复制 Token

点击 “Continue to summary” → “Create Token”，然后复制生成的 Token 字符串（以 abc123... 开头的一长串）。

5

添加到本应用

打开应用 → 点击右上角 “添加账户” → 填写：

账户名称 随便填，例如“我的账户”
邮箱 / 描述 可选，用于备注
API Token 粘贴刚才复制的 Token

点击 “验证并保存”，如果 Token 有效，应用会自动获取 Account ID 并加载域名列表。

常见问题

Token 验证失败？

请确认 Token 已启用、权限至少包含 帐户 - 帐户（读取）、帐户 - 帐户设置（读取）和区域 - 区域（读取），且资源范围包含你的账户/域名。

某些页面没有数据？

说明 Token 缺少对应权限。例如 Workers 页面需要 帐户 - Workers 脚本（编辑），R2 页面需要 帐户 - Workers R2 存储（编辑）。

应用打不开或白屏？

请确保系统已安装 Microsoft Edge WebView2 Runtime。如仍有问题，可尝试卸载后删除 C:\Program Files\cloudflare-manager 目录，再重新安装。
